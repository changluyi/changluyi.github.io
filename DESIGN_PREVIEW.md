## 🎭 动画效果详解

### 1. 页面加载动画

**淡入上移动画 (fadeInUp):**
```css
0% {
  opacity: 0;
  transform: translateY(30px);
}
100% {
  opacity: 1;
  transform: translateY(0);
}
```

**应用顺序：**
1. Hero 标题 (0s 延迟)
2. Hero 描述 (0.2s 延迟)
3. 分类卡片 (0.4s 延迟)

### 2. 渐变背景动画

```css
@keyframes gradientShift {
  0%, 100% {
    background-position: 0% 50%;
  }
  50% {
    background-position: 100% 50%;
  }
}
```

**效果：** 8s 循环，背景渐变缓慢移动

### 3. 卡片悬停动画

```css
/* 悬停前 */
.card {
  transform: translateY(0);
  box-shadow: 0 2px 4px rgba(...);
}

/* 悬停后 (0.3s cubic-bezier) */
.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 12px 24px rgba(...);
}
```

**缓动函数：** `cubic-bezier(0.4, 0, 0.2, 1)`
- 开始稍快
- 中间减速
- 结束平滑

### 4. 链接下划线动画

```css
/* 悬停前 */
a::after {
  width: 0;
}

/* 悬停后 (0.3s ease) */
a:hover::after {
  width: 100%;
}
```

## 🌓 深色模式切换

### 自动切换

使用 `prefers-color-scheme` 媒体查询：

```css
@media (prefers-color-scheme: dark) {
  :root {
    /* 深色模式变量 */
  }
}
```

### 切换效果

**背景：** 浅色 (0.3s) → 深色
**文字：** 深色 (0.3s) → 浅色
**卡片：** 白色 (0.3s) → 深灰蓝

**示例：**
```
浅色模式：
白色背景 + 深色文字 + 浅灰边框

↓ (0.3s 过渡)

深色模式：
深蓝黑背景 + 浅色文字 + 深灰蓝边框
```

## 📊 性能优化效果

### CSS Containment

**优化前：**
```
页面重排时：
浏览器重新计算整个页面布局
```

**优化后：**
```
页面重排时：
浏览器只计算包含元素内部布局
(article, .card, .post-card)
```

**性能提升：** ~30-50% 重排时间减少

### 选择性过渡

**优化前 (全局过渡):**
```css
* {
  transition: all 0.3s ease; /* ❌ 性能差 */
}
```

**优化后 (选择性过渡):**
```css
.theme-transition {
  transition: background-color 0.3s ease,
              color 0.3s ease,
              border-color 0.3s ease; /* ✅ 只过渡必要属性 */
}
```

## 🚀 使用建议

### 1. 渐变使用

**推荐：**
- ✅ Hero 背景
- ✅ 按钮和 CTA
- ✅ 装饰元素
- ✅ 品牌元素

**避免：**
- ❌ 大面积背景 (性能)
- ❌ 文字背景 (可读性)

### 2. 动画使用

**推荐：**
- ✅ 微交互 (悬停、点击)
- ✅ 页面加载
- ✅ 状态转换

**避免：**
- ❌ 自动播放 (可访问性)
- ❌ 过于频繁 (性能)

### 3. 卡片使用

**推荐：**
- ✅ 内容分组
- ✅ 列表项
- ✅ 信息展示

**最佳实践：**
- 保持一致的间距
- 使用适当的阴影
- 确保足够的点击区域
